# 线上优先开发原则

本项目从 2026-06-18 起执行以下原则：

1. Supabase 是正式数据、任务状态、回测结果和模拟交易记录的唯一权威来源。
2. 正式功能必须在线上环境可持续运行，不能依赖某一台电脑上的文件。
3. 本地文件、CSV 和缓存只用于开发测试、首次迁移、故障诊断，不作为产品正式结果。
4. 历史行情采用线上数据库持久化；每日任务只增量更新缺失交易日。
5. 回测、盘中决策和自动模拟交易读取同一套线上行情与策略配置。
6. 不再使用 Excel 作为正式复盘和产品交付模式；正式报告写入线上数据库并由现有页面展示。
7. 不接真实证券账户，只进行研究、历史验证和自动模拟交易。
8. 新功能优先保证数据口径统一、可追溯和可复现，避免增加重复框架与无用代码。
